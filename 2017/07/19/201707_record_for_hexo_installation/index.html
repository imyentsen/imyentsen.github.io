部落格架站紀錄

Jul 19, 2017

希望到芬蘭之後，可以把未來至少兩年的日子當成一場田野。寫部落格就當作督促自己不太嚴謹地做筆記的方式。

我選擇用 Hexo 當作 CMS（content management system） + Github Pages 當作架站平台，之後希望會是一週至少一篇的方式慢慢留下在芬蘭日子的筆記，如果一年有五十二週，兩年下來應該會有一百多篇吧。到時候回頭來看，一定會發現很多事情已經忘記了，感覺會非常有趣ＸＤ

平台的選擇

會選擇 Hexo 當作 CMS（content management system） + GitHub Pages 作為發佈平台，最主要的考量是發佈的方便性。

平常我在電腦上是使用支援 Markdown 的 Bear App 在做筆記。它格式簡潔、方便好用，並且我在調整格式時，手不用離開鍵盤。習慣之後，真的會回不去，所以我不想每次發佈新文章，都還得慢慢在那邊調整格式。

根據這個邏輯，台灣最主流的 Medium、Blogger、Pixnet、Facebook Note 以及常見的各種所見即所得（WYSIWYG）編輯/發布平台就都不太讓人滿意。

我需要的是一個 file-based 的 CMS，讓我可以直接上傳 Bear 輸出的 md. （markdown 原生格式），變成一篇 blog 文章。 Github Pages 提供的靜態網站託管功能正好可以滿足。

Github Pages 可以用的 CMS 很多，Slant 網站上面有相關比較：34 best static site generators as of 2017 - Slant 。選擇 Hexo 作為 CMS 是看上他不需要資料庫，非常輕量、容易上手，然後基本上使用 node.js，這對我來說比較親切。然後作者 tommy351 是台灣人，有一點加分。

（比較大的問題是，我到後來才發現開發社群以華語用戶為主，碰到問題stackoverflow很少討論，我也不習慣開發用語的中文翻譯，要哭了。）

主題

主題我使用 yg-apollo 下去微調，這是一個土耳其人做的，但原型是另一個中國人做的 apollo 。我本來是先用 apollo，它很陽春，但是結構簡單，要調整不難，後來在慢慢找一些想要的功能例如 tags 歸檔頁面等等時，在 apollo 的 fork 清單裡面找到它，想要的剛好都有，就用了。

不過我還是有做一些調整，譬如縮窄原本過寬的網頁寬度、加上 hexo-toc 外掛藉由 markdown 的 # tag 自動生成文章目錄、加上 Disqus 回應插件等等。

最後是想讓每一篇文章都有自己的背景，想讓它們有點像是展開信紙，在某個地方被閱讀的感覺。為此第一次寫了點 jade 的語法（這個主題使用 jade 編譯）。

外掛

我使用的外掛包含以下，經測試與 yg-apollo 相容。事實上他 demo 中一些功能也要使用外掛才跑得出來，但 documentation 沒有說明只得自己找

  • Hexo-Less 用以產生首頁的文章簡介
  • Hexo-toc 用以產生文章目錄
  • Hexo-Post Asset Folder 此外掛Hexo 3 之後的版本已經內建，適用於採 github 作為圖片等資源存放 hosting 的用家。

未來還想做的工作

現在的網站算是堪用了，但希望之後可以做到比 medium 還好的閱讀體驗。需要再慢慢修版面，還有加入以下功能：

  1. 站內搜尋： 這個不難，但現在內容也還不多。
  2. Table of Content： 要做到可以滾動監聽，像 dropbox paper 那樣。這個 medium 也還沒有，但我覺得非常重要。
  3. Inline Comment： 可以逐行回應文章，而非只是下面一個小小區塊。這是 medium 最好用的地方之一
  4. Caption of Image：文章中的照片可以在旁邊顯示 caption。
  5. Markdown 渲染器規則：現在的渲染跟 Bear 原生的規則還是有點不一樣，所以有時會有點差異，希望調整到完全同步。
  6. 想讓文章閱讀的時候，背影照片也有比較慢的滾動效果，所謂的 parallel scrolling。這樣「我在某個地方看到這篇文章」的場景感，會逼真一點點吧？！
